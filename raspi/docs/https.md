# 🏠 Home Network HTTPS Setup - Complete Guide

## 📦 What You'll Get

- ✅ Valid HTTPS certificates for all your services
- ✅ Works on macOS without Bonjour conflicts (using `.home.arpa`)
- ✅ Easy certificate distribution via web pages
- ✅ QR code for quick mobile setup
- ✅ Beautiful setup pages with instructions

## 🚀 Quick Start

### Step 1: Update Your Files

1. **Replace your Caddyfile** with the new version (see artifacts)
2. **Update docker-compose.yml** - Add this line to the Caddy service volumes:
   ```yaml
   - ./www:/config/www:ro
   ```
3. **Update Vaultwarden DOMAIN** in docker-compose.yml:
   ```yaml
   DOMAIN: "https://vaultwarden.home.arpa"
   ```

### Step 2: Create HTML Files

Create a `www` directory in your docker-compose location:
```bash
mkdir -p ./www
```

Save these files in the `www` directory:
- `qr.html` - QR code page for mobile devices
- `setup.html` - Desktop setup instructions page

### Step 3: Run Setup Script

Make the setup script executable and run it:
```bash
chmod +x setup.sh
./setup.sh
```

The script will:
- Create necessary directories
- Extract the certificate from Caddy
- Generate QR code
- Display next steps

### Step 4: Configure Pi-hole DNS

Access Pi-hole at `http://192.168.1.X:8000/admin`

Go to **Local DNS → DNS Records** and add:

| Domain | IP Address |
|--------|------------|
| cert.home.arpa | 192.168.1.X |
| qr.home.arpa | 192.168.1.X |
| setup.home.arpa | 192.168.1.X |
| vaultwarden.home.arpa | 192.168.1.X |
| octoprint.home.arpa | 192.168.1.X |
| pihole.home.arpa | 192.168.1.X |

### Step 5: Configure Devices to Use Pi-hole

**Option A (Recommended):** Enable Pi-hole DHCP
- In Pi-hole: Settings → DHCP → Enable DHCP Server
- Disable DHCP on your router
- Devices will automatically get Pi-hole as DNS

**Option B:** Manual DNS
- Set each device's DNS to your Pi-hole IP
- Or set it in your router's DHCP settings

### Step 6: Install Certificates

**For Desktop/Laptop:**
Visit `http://setup.home.arpa` from any device on your network and follow the instructions.

**For Mobile:**
Visit `http://qr.home.arpa` and scan the QR code, then follow installation steps.

## 🎯 Certificate Installation Quick Reference

### Windows
```powershell
# Download
Invoke-WebRequest -Uri "http://cert.home.arpa" -OutFile "$env:TEMP\caddy-root-ca.crt"

# Install (Admin PowerShell)
Import-Certificate -FilePath "$env:TEMP\caddy-root-ca.crt" -CertStoreLocation Cert:\LocalMachine\Root
```

### macOS
```bash
# Download
curl -o ~/Downloads/caddy-root-ca.crt http://cert.home.arpa

# Install
sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain ~/Downloads/caddy-root-ca.crt
```

### Linux (Ubuntu/Debian)
```bash
# Download
wget http://cert.home.arpa -O caddy-root-ca.crt

# Install
sudo cp caddy-root-ca.crt /usr/local/share/ca-certificates/
sudo update-ca-certificates
```

### Linux (Fedora/RHEL)
```bash
# Download
wget http://cert.home.arpa -O caddy-root-ca.crt

# Install
sudo cp caddy-root-ca.crt /etc/pki/ca-trust/source/anchors/
sudo update-ca-trust
```

### iOS
1. Visit `http://qr.home.arpa` → Scan QR code
2. Settings → Profile Downloaded → Install
3. Settings → General → About → Certificate Trust Settings
4. Enable "Caddy Local Authority"

### Android
1. Visit `http://qr.home.arpa` → Scan QR code
2. Settings → Security → Install from storage
3. Select "CA certificate" → Choose downloaded file

## 🔍 Troubleshooting

### DNS Not Resolving
```bash
# Test DNS
nslookup vaultwarden.home.arpa

# Should return your Pi-hole IP
# If not, check Pi-hole DNS records and device DNS settings
```

### Certificate Not Trusted
1. Verify certificate is installed correctly
2. Clear browser cache
3. Restart browser
4. Try accessing with `https://` (not `http://`)

### Check Caddy Logs
```bash
docker logs caddy
```

### Verify Certificate Exists
```bash
docker exec caddy ls -la /data/caddy/pki/authorities/local/
```

### Regenerate QR Code
```bash
qrencode -t PNG -o ./www/qr-code.png -s 10 "http://cert.home.arpa"
```

## 🎨 Customization

### Change Color Scheme
Edit the gradient in `qr.html` and `setup.html`:
```css
background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
```

### Add More Services
In Caddyfile, add:
```caddy
myservice.home.arpa {
    reverse_proxy myservice:port
    encode gzip
}
```

Then add DNS record in Pi-hole.
